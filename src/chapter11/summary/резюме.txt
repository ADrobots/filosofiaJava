Способы хранения объектов:

1)Массив ассоциирует с объектом числовой индекс. Содержит объекты известного типа.
  Хранит примитивы и может иметь несколько измерений(одномерный, двумерный).
  После создания изменить размер невозможно.

2)Коллекция Collection заполняется одиночными элементами.
  Map хранит ассоциированные пары "ключ-значение".
  При использовании обобщенных типов указывается тип храним объектов,
  при извлечении объектов не требуется преобразование типа. Collection и Map автоматически
  изменяет размер при добавлении объектов. Контейнер не может хранить примитивы, но механизм
  автоматической упаковки обеспечит преобразования к типам -оберткам, хранимым в контейнере.

3)Список(List), подобно массиву, ассоциирует с объектами числовой индекс

4)Для частого получения произвольных объектов нужно использовать список ArrayList.
  Для частой вставки и удаления в середине списка, лучше использовать списко LinkedList

5)Класс LinkedList предоставляет возможность очередей и стеков

6)Карта Map ассоциирует с объектами не только числа, но и другие объекты.
  Класс HashMap используется для быстрого доступа к элементам.
  Класс TreeMap хранит набор ключей в отсортированом виде, уступая по скорости HashMap
  Класс LinkedHashMap упорядочивает элементы в порядке добавления, обеспечивает быстрый доступ
  благодаря хэшированию.

7)Множество Set хранит не более одного экземпляра определенного объекта.
  Множество HashSet используется для усроренного поиска,
  Множество TreeSet используется для хранения элементов в отсортированном виде,
  Множество LinkedHashSet используется для храниения элементо в в порядке добавления.
